Decision: Commands fall into four categories

- content:
  Produces user-facing content (AI answer or raw content)
  Examples: chat, web-chat, file-chat, read-file

- report:
  Produces structured human-readable summaries
  Examples: status, doctor, stats, sessions, web-search

- artifact:
  Creates or previews stored artifacts
  Examples: web-fetch

- action:
  Mutates or cleans system state
  Examples: clear, summarize, web-cleanup, repair, migrate

Decision: Source is a concept, not a module

Current source types:
- web artifacts (stored JSON from fetch/search)
- local files (explicit read-file/file-chat)
- session memory (stored conversation state)

Commands may use one or more sources.

The concept of “source” is intentionally not abstracted into a shared module yet.
It will only be formalized when a second interface (UI/shell/API) requires it.
